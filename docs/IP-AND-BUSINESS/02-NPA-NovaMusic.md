## BACKGROUND OF THE INVENTION

### Field of the Invention

The present invention relates to computer-implemented systems and methods for artificial intelligence-driven music mixing and DJ automation, real-time crowd engagement quantification and adaptive response, browser-based spatial audio processing, artist-to-DJ direct music licensing with automated royalty computation, peer-to-peer audio distribution, and multi-device synchronized visual experiences — together constituting a unified Social AI Music Experience platform for live and remote entertainment events. The invention integrates these systems into a single browser-based application operable without installation of native software.

### Background Art

#### 1. The Fragmented Digital Music Performance Landscape

Digital music creation, performance, and distribution have evolved through three largely separate technological lineages that existing platforms have failed to integrate:

**Professional DJ Software:** Desktop-native applications (Serato DJ Pro, Native Instruments Traktor, Pioneer rekordbox) provide sophisticated audio mixing capabilities including multi-deck playback, hardware controller integration, waveform visualization, and effects processing. These platforms are designed for professional hardware-connected operation and are inaccessible to casual users. Critically, they provide no crowd engagement infrastructure, no music marketplace, no AI planning assistance, and no audience management tools. They function as instruments, not platforms.

**Music Streaming Services:** Consumer platforms (Spotify, Apple Music, Amazon Music, Tidal) provide licensed access to vast music catalogs for passive listening. They implement AI recommendation algorithms (Spotify's Discover Weekly, Daily Mix), mood-based curation, and social features (shared playlists, friend activity). However, they provide zero DJ functionality, no live event management, no audience engagement layer, no artist-direct marketplace, and no professional audio processing. They are distribution channels, not performance platforms.

**Live Event Technology:** Venue management and audience engagement platforms (Eventbrite, StageIt, Veeps, MangoJam) provide ticketing, streaming, and fan interaction tools but lack integrated music processing, DJ functionality, or artist marketplace features. Fan engagement is typically limited to text chat with no real-time music-synchronized interaction.

The NovaMusic invention integrates all three lineages — professional DJ audio processing, AI music intelligence, and real-time crowd engagement — into a single browser-based platform, creating a new category the inventors designate the "Social AI Music Experience."

### Embodiment 2: AI DJ Auto-Mix Engine — Detailed Implementation

#### 2.1 Track Analysis Pipeline

The track analysis pipeline processes up to 50 simultaneously uploaded audio files. Each file is processed as follows:

**Stage 1 (Server-Side Extraction):** The `music-metadata` package parses the audio file's binary headers to extract: `format.duration` (seconds), `common.title`, `common.artist`, `common.album`, `common.bpm` (if present in ID3v2 tags), and `format.container`. Missing fields default to reasonable values (BPM: 128, key: "C major", energy: 0.7, genre: "Electronic").

**Stage 2 (Client Energy Transmission):** The browser independently decodes the audio file via `AudioContext.decodeAudioData()`, computes RMS energy over the decoded buffer, and transmits the computed energy percentage in the multipart form upload alongside the binary audio data.

**Stage 3 (LLM Inference):** The following structured prompt is submitted to the GPT-4.1-mini API for each track:

```
Analyze this music track and provide BPM, key, energy, genre, and mood.
Track: {title} by {artist}

Genre-specific BPM ranges for reference:
- Amapiano: 112-114 BPM
- Phonk: 130-160 BPM
- UK Garage: 130-136 BPM
- EDM/House: 125-135 BPM
- Hip Hop/Trap: 70-100 BPM
- Afrobeats: 95-115 BPM
- Dancehall: 80-100 BPM
- Drill: 135-145 BPM
[... 15 genres total ...]

Return JSON: { bpm, key, energy, genre, mood, isTrending: false }
```

**Stage 4 (Trend Verification):** A timeout-gated HTTP GET request is made to `api.duckduckgo.com/?q={title}+{artist}+music&format=json`. The `Abstract` field of the response is matched against the keywords: "billboard", "spotify", "grammy", "platinum", "trending", "chart", "number one", "#1". If any keyword matches, `isTrending = true` is set with the matched Abstract text as evidence.

**Data Reconciliation:** Final field resolution applies source priority:
- `energy`: client-computed > server-default (0.7)
- `bpm`: server-extracted ID3 > AI-inferred
- `key`: AI-inferred > server-default
- `genre`: AI-inferred > server-default
- `isTrending`: web search result (overrides AI default of false)

#### 2.2 Harmonic Compatibility Score Computation

For each ordered pair of tracks (outgoing track T_out, incoming track T_in):

```
BPM_SCORE:
  diff = |T_out.bpm - T_in.bpm|
  ratio = min(T_out.bpm, T_in.bpm) / max(T_out.bpm, T_in.bpm)
  
  if diff <= 2: bpmScore = 50
  elif diff <= 5: bpmScore = 40
  elif diff <= 8: bpmScore = 25
  elif diff <= 12: bpmScore = 15
  else: bpmScore = 0
  
  if 0.48 <= ratio <= 0.52: bpmScore += 18  // half-time/double-time bonus

HARMONIC_SCORE:
  camelotOut = CAMELOT_MAP[T_out.key]  // maps 24 keys to Camelot codes 1A-12B
  camelotIn = CAMELOT_MAP[T_in.key]
  
  if camelotOut == camelotIn: harmonicScore = 30
  elif same_number(camelotOut, camelotIn): harmonicScore = 15  // relative major/minor
  elif adjacent_number(camelotOut, camelotIn) AND same_mode: harmonicScore = 20
  else: harmonicScore = 0

totalScore = bpmScore + harmonicScore  // range: 0-100

BLEND_TYPE:
  if totalScore >= 75: blendType = "long-blend", beatDuration = 32
  elif totalScore >= 55: blendType = "quick-blend", beatDuration = 16
  elif totalScore >= 35: blendType = "echo-out", beatDuration = 8
  else: blendType = "cut", beatDuration = 4

RATE_ADJUSTMENT:
  if diff <= 12: rateMultiplier = T_out.bpm / T_in.bpm
  else: rateMultiplier = 1.0  // no tempo adjustment for large differences
```

#### 2.3 Optimal Set Construction Algorithm

```
INPUT: tracks[] (AnalyzedTrack array), vibe (chill | hype | balanced)

if vibe == "chill": return tracks.sortBy(energy, ascending)
if vibe == "hype": return tracks.sortBy(energy, descending)

// Balanced: greedy nearest-neighbor
remaining = Set(tracks)
ordered = []
current = tracks.minBy(t => |t.bpm - 120|)  // seed: closest to 120 BPM
ordered.push(current)
remaining.delete(current)

while remaining.size > 0:
  nextTrack = remaining.maxBy(t => scoreMixTransition(current, t))
  ordered.push(nextTrack)
  remaining.delete(nextTrack)
  current = nextTrack

return ordered
```

#### 4.2 Algorithmic Impulse Response Generation

```typescript
function createImpulseResponse(ctx: AudioContext, duration: number, decay: number): AudioBuffer {
  const sampleRate = ctx.sampleRate;
  const length = sampleRate * duration;
  const impulse = ctx.createBuffer(2, length, sampleRate);
  
  for (let channel = 0; channel < 2; channel++) {
    const data = impulse.getChannelData(channel);
    for (let i = 0; i < length; i++) {
      data[i] = (Math.random() * 2 - 1) * Math.pow(1 - i / length, decay);
    }
  }
  return impulse;
}
```

#### 4.3 Chromagram-Based Key Detection

```typescript
const A_MAJOR_PROFILE = [6.35, 2.23, 3.48, 2.33, 4.38, 4.09, 2.52, 5.19, 2.39, 3.66, 2.29, 2.88];
const A_MINOR_PROFILE = [6.33, 2.68, 3.52, 5.38, 2.60, 3.53, 2.54, 4.75, 3.98, 2.69, 3.34, 3.17];

function detectKey(audioBuffer: AudioBuffer): { key: string; camelot: string } {
  const sampleRate = audioBuffer.sampleRate;
  const channelData = audioBuffer.getChannelData(0);
  const chroma = new Float32Array(12).fill(0);
  const samplesPerSecond = Math.floor(sampleRate / 10);
  const analysisLength = Math.min(30 * sampleRate, audioBuffer.length);
  
  for (let sample = 0; sample < analysisLength; sample += samplesPerSecond) {
    for (let midiNote = 36; midiNote <= 84; midiNote++) {
      const freq = 440 * Math.pow(2, (midiNote - 69) / 12);
      let correlation = 0;
      const period = Math.floor(sampleRate / freq);
      for (let k = 0; k < period && sample + k + period < channelData.length; k++) {
        correlation += channelData[sample + k] * channelData[sample + k + period];
      }
      const pitchClass = midiNote % 12;
      chroma[pitchClass] += Math.abs(correlation);
    }
  }
  
  const maxChroma = Math.max(...chroma);
  const normalizedChroma = chroma.map(v => v / maxChroma);
  
  let bestKey = "C"; let bestScore = -Infinity; let bestIsMinor = false;
  for (let transpose = 0; transpose < 12; transpose++) {
    const majorCorr = correlate(normalizedChroma, rotate(A_MAJOR_PROFILE, transpose));
    const minorCorr = correlate(normalizedChroma, rotate(A_MINOR_PROFILE, transpose));
    if (majorCorr > bestScore) { bestScore = majorCorr; bestKey = NOTES[transpose]; bestIsMinor = false; }
    if (minorCorr > bestScore) { bestScore = minorCorr; bestKey = NOTES[transpose]; bestIsMinor = true; }
  }
  
  const keyName = `${bestKey} ${bestIsMinor ? "minor" : "major"}`;
  return { key: keyName, camelot: CAMELOT_MAP[keyName] };
}
```

#### 4.4 Crossfader Curve Implementations

```typescript
function updateCrossfade(crossfadeValue: number, mode: "smooth" | "club" | "cut") {
  // crossfadeValue: 0.0 (full A) to 1.0 (full B)
  let volA: number, volB: number;
  
  switch (mode) {
    case "smooth":  // Linear
      volA = 1 - crossfadeValue;
      volB = crossfadeValue;
      break;
    case "club":   // Equal-power (constant energy)
      volA = Math.sin((1 - crossfadeValue) * Math.PI / 2);
      volB = Math.sin(crossfadeValue * Math.PI / 2);
      break;
    case "cut":    // Hard-cut (scratch mixing)
      volA = crossfadeValue < 0.5 ? 1 : 0;
      volB = crossfadeValue > 0.5 ? 1 : 0;
      break;
  }
  
  deckA.gain.setTargetAtTime(volA, ctx.currentTime, 0.003);
  deckB.gain.setTargetAtTime(volB, ctx.currentTime, 0.003);
}
```

export const PLATFORM_CUT = 0.15;

// Monthly royalty calculation:
async function calculateMonthlyRoyalties(period: string) {
  const playEvents = await db.select().from(play_events)
    .where(and(
      gte(play_events.playedAt, periodStart),
      lte(play_events.playedAt, periodEnd),
      eq(play_events.licenseType, "royalty")
    ));
  
  const byArtist = groupBy(playEvents, e => e.artistId);
  
  for (const [artistId, events] of Object.entries(byArtist)) {
    const totalPlays = events.length;
    const totalAmount = events.reduce((sum, e) => sum + e.royaltyAmount, 0);
    const platformFee = totalAmount * PLATFORM_CUT;
    const netAmount = totalAmount * DJ_CUT;
    
    await db.insert(royalty_payouts).values({
      artistId, period, totalPlays, totalAmount,
      platformFee, netAmount, status: "pending"
    });
  }
